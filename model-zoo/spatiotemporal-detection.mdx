Spatio-temporal action detection is the task of simultaneously locating actors in video frames (using bounding boxes) and classifying what action each actor is performing. Unlike temporal action localization, which only finds _when_ an action occurs, spatio-temporal detection answers _who_, _where_, and _what_ at each moment in a video. MMAction2 implements this using a two-stage pipeline: a video feature extractor coupled with a region-based detection head that classifies actor proposals.

## Task overview

MMAction2's detection pipeline follows the approach used in the AVA benchmark:

1. **Person detection** — An off-the-shelf person detector (Faster R-CNN from MMDetection) identifies bounding boxes in each keyframe.
2. **Video feature extraction** — A 3D video backbone extracts spatio-temporal features for the video clip surrounding each keyframe.
3. **RoI pooling + classification** — Features within each detected bounding box are pooled and passed to a multilabel action classifier.

This two-stage design allows the detection head to be trained independently of the person detector, and makes it straightforward to swap video backbones.

<Note>
Spatio-temporal detection in MMAction2 requires [MMDetection](https://github.com/open-mmlab/mmdetection) for the person detection stage. Install it with `mim install mmdet`.
</Note>

  <Tab title="ACRN">
    ### ACRN — Actor-Centric Relation Network (ECCV 2018)

    ACRN augments an existing detection backbone with an actor-centric relational module that computes pairwise relations between each detected actor and all other actors and scene elements in the frame. The relation features are accumulated and concatenated to the actor's RoI feature before classification. ACRN improves performance on actions that require understanding interactions (e.g., "hand shake", "embrace").

    **AVA 2.1 results (SlowFast ResNet-50, K400 pretrain, 8×8×1):**

    | Dataset | mAP |
    |---|---|
    | AVA 2.1 | 27.65 |
    | AVA 2.2 | 27.71 |
  </Tab>
  <Tab title="LFB">
    ### LFB — Long-Term Feature Bank (CVPR 2019)

    LFB augments a short-term action recognition backbone with a long-term feature bank that stores features from all other clips in the same video. A non-local attention operator retrieves the most relevant historical and future context to support the current clip's classification, enabling temporal reasoning over the full video duration at inference time.

    **Supported datasets:** AVA 2.1
  </Tab>
  <Tab title="VideoMAE">
    ### VideoMAE (NeurIPS 2022)

    VideoMAE-pretrained ViT backbones can be fine-tuned for spatio-temporal action detection on AVA-Kinetics. The masked autoencoder pre-training provides rich spatio-temporal representations that transfer well to the detection task.

    **AVA 2.2 results (trained on AVA-Kinetics):**

    | Backbone | Pretrain | Sampling | mAP |
    |---|---|---|---|
    | ViT-Base | Kinetics-400 | 16×4×1 | 33.6 |
    | ViT-Large | Kinetics-400 | 16×4×1 | 38.7 |
  </Tab>

## Running detection training and inference

### Training

```bash
python tools/train.py \
  configs/detection/slowfast/slowfast_kinetics400-pretrained-r50_8xb16-4x16x1-20e_ava21-rgb.py \
  --seed 0 --deterministic
```

### Evaluating a checkpoint

```bash
python tools/test.py \
  configs/detection/slowfast/slowfast_kinetics400-pretrained-r50_8xb16-4x16x1-20e_ava21-rgb.py \
  checkpoints/slowfast_ava21.pth \
  --dump result.pkl
```

## Data preparation

Spatio-temporal detection requires:

1. **Keyframe extraction** — Extract the center frame for each 1-second interval in AVA videos.
2. **Pre-computed person boxes** — Obtain person detections from a trained Faster R-CNN detector and store them in the AVA proposal format.
3. **Video clips** — Store video clips around each keyframe for feature extraction.

Refer to the AVA, AVA-Kinetics, and MultiSports data preparation guides in `tools/data/` for dataset-specific instructions.

## Evaluation metrics

| Metric | Description |
|---|---|
| mAP@0.5 | Mean average precision at temporal IoU threshold 0.5 (standard AVA metric) |
| f-mAP | Frame-level mAP: evaluates detection results independently at each frame |
| v-mAP@T | Video-level mAP: uses 3D IoU to evaluate full action tubes at threshold T |

<Tip>
MultiSports uses both f-mAP and v-mAP. For fair comparison against published methods, report both metrics at the thresholds specified in the MultiSports competition.
</Tip>
